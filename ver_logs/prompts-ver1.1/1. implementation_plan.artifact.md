# Implementation Plan - Unified Multi-Tiered Filtering System

Implement a comprehensive filtering system for Seika to ensure an adult-content-free experience and provide granular control over extensions.

## User Review Required

- **Whitelist Management**: I propose adding a dedicated "Extension Whitelist" screen under Browse Settings to manage approved extensions when in `ALLOW_SELECTED` mode.
- **Strict Content Filter**: The default set of keywords for `Manga.isAdultContent()` will be `["Pornographic", "Hentai", "Adult", "Smut", "NSFW"]`. These will be case-insensitive and checked against manga genres/tags.
- **Blocking Behavior**: When an extension is blocked, it will be completely hidden from all lists, and any attempt to access its content (via deep-link or history) will show a "Content unavailable" message.

Define the core logic for identifying adult content and store user filter preferences.

#### [Manga.kt](file:///C:/Users/gsmur/Documents/GitHub/Seika/domain/src/main/java/tachiyomi/domain/manga/model/Manga.kt)

```kotlin
fun Manga.isAdultContent(): Boolean {
    val adultTags = listOf("Pornographic", "Hentai", "Adult", "Smut", "NSFW")
    return genre?.any { tag ->
        adultTags.any { it.equals(tag, ignoreCase = true) }
    } ?: false
}
```
